# 📘 2. TasMind 웹 대시보드 상세 사용 설명서 (User Manual)

**TasMind**는 Tasmota 펌웨어가 탑재된 IoT 디바이스들을 AI 기반 자연어로 제어, 모니터링, 관리하고, 복잡한 자동화 시나리오 및 워크플로우를 쉽게 구성할 수 있는 통합 게이트웨이 플랫폼입니다. 

본 사용 설명서는 사용자가 TasMind 웹 대시보드의 주요 기능과 UI 구성 요소를 효과적으로 활용할 수 있도록 작성되었습니다.

## 1. 시작하기 & 계정 인증

TasMind는 사용자의 개인정보(이메일, 이름 등)를 서버에 저장하지 않는 **보안 우선 설계**를 지향합니다.

> [!NOTE]
> **개인정보 보호**: Google 로그인 시 제공되는 Google ID의 **SHA256 해시값**만을 사용자 식별용으로 활용하므로 개인정보 유출 위험이 없습니다.

### 🔑 회원가입 및 로그인 절차
1. **Google 1-Click 로그인**:
   - 로그인 화면에서 **`Sign in with Google`** 버튼을 클릭합니다.
2. **회원가입 동의 및 닉네임 설정**:
   - 처음 로그인한 경우 회원가입 화면으로 이동합니다.
   - 서비스에서 사용할 **닉네임(2~30자)**을 입력합니다.
   - `[필수] 이용약관` 및 `[필수] 고유 식별자 활용 동의` 항목 체크 후 **`Google 계정으로 가입`**을 진행합니다.
3. **관리자 승인 (Pending Status)**:
   - 보안 강화를 위해 가입 신청 후 관리자의 승인이 필요합니다.
   - 승인 전에는 **승인 대기 중(Pending Approval)** 화면이 표시되며, 승인이 완료되면 즉시 대시보드 기능을 이용할 수 있습니다.

## 2. 대시보드 화면 구성 (Layout & Header Controls)

TasMind 웹 대시보드는 최신 **Bento Grid** 레이아웃을 채택하여 모바일과 데스크톱 환경 모두에서 직관적인 조작을 지원합니다.

### 🌐 상단 헤더 컨트롤 (Header Controls)
- **도메인 선택 (Domain Selector)**:
  - `🏠 Smart Home`, `🌱 Smart Farm`, `🏭 Smart Factory`, `🏢 Smart Building` 중 운영 환경에 맞는 도메인을 선택하여 맞춤 시나리오 및 카테고리를 활용할 수 있습니다.
- **다국어 변경 (Language Selector)**:
  - `🌐 한국어` / `🌐 English` 간 실시간 언어 전환을 지원합니다.
- **다크/라이트 테마 토글 (Theme Toggle)**:
  - ☀️ 해 / 🌙 달 아이콘 버튼으로 시각적 쾌적함에 맞게 테마를 변경합니다.
- **사용자 프로필 & 등급**:
  - 현재 로그인한 사용자 닉네임 및 사용자 권한 Badge(`Admin` 또는 일반 사용자 등급)가 표시됩니다.

## 3. 주요 탭별 세부 기능 가이드

### 3.1. 홈 뷰 (Home)

홈 뷰는 전체 스마트 디바이스의 현황과 주요 센서 지표, AI 대화 창 및 간편 제어 기능을 한눈에 파악할 수 있는 메인 대시보드입니다.

#### 1) KPI 메트릭 카드 (Customizable KPI Cards)
- **온라인 기기 수 (Online Devices)**: 현재 게이트웨이에 정상 연결된 기기 수
- **평균 온도 / 평균 습도**: 등록된 환경 센서들의 실시간 평균값
- **실시간 소비전력 (Active Power)** / **오늘 누적 사용량 (Today's Energy)**: 전력 측정 모듈(POW) 센서 데이터
- **활성 경고 (Active Alerts)**: 온습도 이상 또는 오프라인 상태 경고 알림 수

#### 2) 실시간 환경 모니터 (Real-time Environment Monitor)
- 주요 방/구역별 온습도 및 조도 등 최신 센서 상태를 실시간 스파크라인 카드 형태로 표시합니다.

#### 3) 간편 제어 시나리오 (Quick Control Scenes)
- 원클릭으로 여러 기기를 동시에 제어하는 시나리오입니다.
- **예시**: `외출(Outing)`, `귀가(Welcome)`, `취침(Sleep)`, `주간 송풍`, `야간 보온`, `강제 급수` 등 버튼 하나로 즉시 동시 제어 명령이 전송됩니다.

#### 4) AI Agent 대화 창 (Interactive AI Chat)
- **자연어 제어**: 입력창에 *"거실 조명 켜줘"*, *"온실 습도 알려줘"* 처럼 대화하듯 명령을 입력합니다.
- **Quick Actions**: 자주 사용하는 추천 명령어가 칩 형태로 제공되어 원클릭으로 명령을 전송할 수 있습니다.
- **연결 상태 표시**: AI 서버와의 WebSocket 통신 상태(`Connected` / `Disconnected`)를 실시간으로 보여줍니다.

#### 5) 가상 노드 & 시나리오 현황 (Virtual Node & Scenarios)
- 물리 기기들을 방(거실, 안방, 온실 등) 단위로 그룹화한 **가상 노드(Virtual Node)**와 바인딩된 자동화 시나리오 링키지 현황을 모니터링합니다.

## 3.2. 장치 관리 뷰 (Devices)

연결된 모든 Tasmota 물리 하드웨어 기기와 센서, 릴레이 스위치를 정밀하게 관리하고 설정하는 페이지입니다.

#### 1) 카테고리 필터 & 검색
- `Switch`, `Sensor`, `Shutter` 등 디바이스 유형별 칩 필터링과 실시간 검색창을 이용해 특정 기기를 빠르게 찾을 수 있습니다.

#### 2) 장치 추가 (Add Device)
- **`+ Add Device`** 버튼 클릭 후 등록할 Tasmota 기기의 **Topic ID**를 입력합니다.
- 팝업창으로 전달되는 **1회성 MQTT 암호** 및 서버 설정 도메인을 확인하여 Tasmota 기기의 MQTT 설정에 입력합니다.

> [!WARNING]
> 발급된 MQTT 비밀번호는 보안상 창을 닫으면 다시 확인할 수 없으므로 반드시 복사하여 보관하세요.

#### 3) 장치 세부 설정 사이드 시트 (Detail Side Sheet)
장치 카드를 클릭하면 우측에서 상세 설정 패널이 열립니다.
- **닉네임 및 한글 명칭 설정**: 기기 본체, 스위치 릴레이 채널(`Power1`, `Power2`), 센서(`AM2301` 등)에 사용자 친화적인 한글 별명을 지정합니다.
- **정전 복구 전원 모드 (PowerOnState)**: 정전 후 전기가 다시 들어왔을 때 전원 상태(`Off`, `On`, `이전 상태 기억`)를 설정합니다.
- **텔레메트리 보고 주기 (TelePeriod)**: 센서 데이터 수집 주기를 설정합니다 (기본 300초 = 5분).
- **블라인드 / 셔터 정밀 교정 (Shutter Control)**:
  - 셔터 릴레이 매핑 및 차고문 모드/일반 셔터 모드 선택
  - 완전 열림(`ShutterOpenDuration`) / 완전 닫힘(`ShutterCloseDuration`) 소요 시간(초 단위) 설정
  - 0%~100% 개방 비율 정밀 제어

#### 4) 센서 데이터 실시간 차트 (Sensor Data Charts)
- 조회할 기기와 센서 유형, 시간 범위(`1H`, `6H`, `24H`, `7D`)를 선택하여 시간에 따른 변화 추이를 차트로 분석할 수 있습니다.

## 3.3. 규칙 & 자동화 뷰 (Rules)

기기의 센서 조건, 시간 스케줄, 상태 변화에 따라 자동으로 작동하는 자동화 규칙을 관리합니다.

#### 1) 룰 실행 방식 구조
- **Tasmota 물리 규칙 (Tasmota Local Rule - Slot 1~3)**:
  - Tasmota 기기 메모리에 MQTT로 직접 구문을 업로드하여, 게이트웨이 **인터넷 연결이 끊긴 오프라인 상태에서도 기기 자체 CPU가 자율 동작**합니다.
- **서버 룰 엔진 (Server Rule Engine)**:
  - 다중 기기 연동이나  complex 조건 처리 시 서버의 스케줄러 및 데이터베이스에서 조건을 감지하여 제어합니다.

#### 2) 실행 모드 (Execution Modes)
- **`repeat`**: 조건이 충족되어 있는 동안 지정된 액션을 계속 실행합니다.
- **`once`**: 조건이 참이 될 때 1회만 실행한 후 대기합니다.
- **`toggle`**: 조건 충족 시 액션을 실행하고, **조건이 해제되면 이전 원상태로 자동으로 복원(Rollback)**합니다.

## 3.4. 개인 설정 뷰 (Settings)

개인화 옵션 및 시스템 구성 상태를 관리합니다.

- **사용자 타임존 (User Timezone)**: `Asia/Seoul (KST)`, `UTC`, `Europe/Berlin` 등 사용자 지역 기준 시간대를 선택합니다. (모든 시나리오 스케줄 및 타임스탬프에 반영됩니다.)
- **회원 등급 & 토큰 사용량 (Token Usage)**:
  - 오늘 / 이번 주 / 이번 달 AI 모델 대화 토큰 소비 현황을 그래프와 수치로 확인합니다.
  - 일일 한도가 초과된 경우 **`한도 늘리기 요청`** 버튼을 통해 업그레이드를 신청할 수 있습니다.
- **KPI 관리 (KPI Management)**: 홈 뷰의 6개 KPI 카드 중 화면에 표시할 카드를 선택적으로 켜고 끌 수 있습니다.
- **회원 탈퇴 (Withdraw Membership)**: 서비스 이용 중단 시 연동 데이터 및 등록 정보를 즉시 삭제 처리합니다.

## 4. AI 챗봇 자연어 명령어 가이드

AI Agent에 자연어로 대화하듯 명령하면, AI가 의도를 분석하여 기기를 정밀 제어합니다.

| 분류 | 대화 예시 (자연어 입력) | AI 내부 동작 |
| :--- | :--- | :--- |
| **기기 별명 설정** | `"test_tasmota 장치에 연결된 AM2301 센서 이름을 안방센서로 지정해줘"` | 센서 닉네임 DB 등록 |
| **가상 노드 생성** | `"Homecontrol 노드를 만들고 tasmota_ai2와 ai4 장치 추가해줘"` | 그룹 노드 생성 및 기기 매핑 |
| **기본 전원 제어** | `"하우스1에 연결된 거실등 켜줘"` | `cmnd/하우스1/POWER1 = ON` |
| **텔레메트리 설정** | `"하우스1 장치의 상태 보고 주기를 5분으로 변경해줘"` | `cmnd/하우스1/TelePeriod = 300` |
| **Backlog 다중 명령** | `"하우스1 전원1을 켜고, 10초 대기한 다음 전원1을 끄는 명령을 백로그로 보내줘"` | `Backlog Power1 ON; Delay 100; Power1 OFF` |
| **셔터 / 블라인드** | `"하우스1 셔터1이 열리는 데 12.5초, 닫히는 데 11.2초 걸리도록 맞춰줘"` | `ShutterOpenDuration1 12.5; ShutterCloseDuration1 11.2` |
| **셔터 위치 이동** | `"하우스1 셔터1을 50% 위치로 이동시켜 줘"` | `ShutterPosition1 50` |
| **시나리오 생성** | `"오전 4시~11시 사이 온도가 28도 이상이고 습도가 50% 이하이면 분무기 켜는 시나리오 만들어줘"` | 복합 조건 DSL 템플릿 컴파일 및 룰 생성 |

## 5. 안전한 이용을 위한 필수 주의사항

> [!CAUTION]
> Tasmota 펌웨어 기기의 물리적 손상 및 오작동 방지를 위해 아래 항목을 반드시 준수해 주세요.

1. **Backlog 명령어 조합 시 `PulseTime` 대신 `Delay` 사용**:
   - 백로그 구문 사용 시 복잡한 계산 공식이 요구되는 `PulseTime` 대신 **0.1초 단위의 `Delay` 명령어**를 사용하세요.
   - *예시*: `Backlog Power1 ON; Delay 100; Power1 OFF` (10초 대기 후 종료)
2. **모터/셔터 릴레이 설정 전 상호 잠금(Interlock) 설정 필수**:
   - 셔터, 블라인드 모터는 정방향과 역방향 릴레이에 동시에 전원이 공급되면 **모터가 타버릴 위험**이 있습니다.
   - 셔터 릴레이 매핑 전 반드시 상호 잠금을 설정하세요. (*예시*: `Interlock 1,2`)
3. **복합 조건 시나리오의 `and` 그룹화**:
   - 시간 범위 조건과 온습도 조건이 결합된 경우, 하나의 조건 블록 안에 `and` 항목으로 묶어 작성하여 오작동(예: 밤시간 불필요 가동)을 예방하세요.
4. **Tasmota 물리 룰 슬롯 한계 (Rule1~Rule3)**:
   - 기기 내부에 직접 저장 가능한 룰은 최대 3개 슬롯(각 511자 제한)입니다. 초과하는 룰은 서버 룰 엔진으로 자동 전환됩니다.
