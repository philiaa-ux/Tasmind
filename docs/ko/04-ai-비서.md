# 4. AI 에이전트 (AI Chat) 사용법과 예시  
# AI 채팅 활용 가이드 - 대화형 스마트 제어

본 가이드는 **TasMind의 AI 채팅**을 효과적으로 활용하는 방법을 안내합니다.  
**웹/앱 대시보드**는 단일 장치의 간단한 제어에, **AI 채팅**은 복합 명령이나 다중 장치 연동, 데이터 분석 등에 더 효율적입니다.

---

## 🔑 기기 식별 기본 개념

TasMind에서는 **장치 ID**와 **닉네임**을 모두 사용할 수 있습니다.

| 구분 | 설명 | 예시 |
|------|------|------|
| **장치 ID** | 시스템 내부에서 사용하는 고유 식별자 | `tasmota_1234`, `esp32_5678` |
| **닉네임 (별명)** | 사용자가 직관적으로 부여한 이름 | `거실`, `안방`, `온실1` |

> 💡 **닉네임을 부여하면** 긴 장치 ID 대신 **"거실의 온도 알려줘"** 처럼 자연스러운 명령이 가능합니다.  
> 장치 ID는 내부 식별자이지만, 닉네임은 **사용자 친화적**으로 자유롭게 설정할 수 있습니다.

---

## 💡 AI 채팅 vs 대시보드 - 언제 무엇을 사용할까?

| 구분 | 웹/앱 대시보드 | AI 채팅 |
|------|---------------|----------|
| **단일 ON/OFF 제어** | ✅ 클릭 한 번 | ⚠️ 타이핑 필요 |
| **여러 장치 제어** | ❌ 각각 클릭 필요 (번거로움) | ✅ 한 문장으로 처리 가능 |
| **센서 데이터 조회** | ❌ 각 센서 클릭 필요 | ✅ 자연어로 한 번에 요청 |
| **데이터 트렌드 분석** | ❌ 그래프 설정 필요 | ✅ 요약/분석까지 제공 |
| **복합 자동화 설정** | ❌ 여러 메뉴 이동 필요 | ✅ 대화로 한 번에 설정 |
| **Tasmota 고급 기능** | ❌ 콘솔 명령어 필요 | ✅ 자연어로 처리 |
| **토큰 효율성** | - | ✅ 복합 명령 시 토큰 절약 |

> 💡 **핵심 팁**: "한 번에 여러 명령이나 복합 명령을 처리할 때는 AI 채팅이 훨씬 효율적입니다. 여러 번의 클릭 대신 한 문장으로 모든 작업을 완료할 수 있어 토큰 사용량도 절약됩니다."

## 🗣️ 실제 대화 예시 모음

### 📋 1. 기기 및 센서 이름 관리

#### 1.1 센서 이름 지정

> **👤 사용자**
> ```
> [장치ID] 장치의 AM2301 센서 이름을 '침실온도센서'로 지정해줘
> ```

> **🤖 AI 응답**
> ```
> ✅ 센서 이름이 성공적으로 변경되었습니다!
> 
> 장치: [장치ID] (내부 ID: tasmota_xxxx)
> 센서: AM2301 → 침실온도센서
> 
> 💡 이제 "침실온도센서의 온도 알려줘"라고 명령할 수 있습니다.
> ```

---

#### 1.2 장치 이름/별명 설정

> **👤 사용자**
> ```
> 장치 ID '[장치ID]'의 이름은 '2층'으로 지정해줘
> ```

> **🤖 AI 응답**
> ```
> ✅ 장치 이름이 변경되었습니다!
> 
> 이전 장치 ID: [장치ID]
> 새 장치 이름(닉네임): 2층
> 
> 💡 이제 "2층의 온도 알려줘" 또는 "2층의 [스위치닉네임] 켜줘"로 명령할 수 있습니다.
> ```

---

#### 1.3 스위치 채널 이름 지정

> **👤 사용자**
> ```
> [장치ID]에 연결된 power4 스위치의 이름을 출입구등으로 지정해줘
> ```

> **🤖 AI 응답**
> ```
> ✅ 스위치 별명이 설정되었습니다!
> 
> 장치: [장치ID]
> 채널: POWER4 → 출입구등
> 
> 💡 이제 "출입구등 켜줘" 또는 "출입구등 꺼줘"로 제어할 수 있습니다.
> ```

#### 2.3 누락된 닉네임 찾기

> **👤 사용자**
> ```
> 현재 등록된 기기 중에서 닉네임이 없는 것들을 모두 찾아줘
> ```

> **🤖 AI 응답**
> ```
> 🔍 닉네임이 없는 장치 및 센서 목록:
> 
> [장치] 닉네임 없음:
> - [장치ID_1] (추천: 차고, 외부창고)
> - [장치ID_2] (추천: 정원, 화단)
> 
> [센서] 닉네임 없음:
> - [장치닉네임] > AM2301-02 (추천: 거실센서, 주방센서)
> - [장치ID] > AM2301-04 (추천: 안방센서, 서재센서)
> 
> [스위치] 닉네임 없음:
> - [장치닉네임] > POWER2 (추천: 현관등, 복도등)
> - [장치ID] > POWER1 (추천: 거실등, 메인조명)
> 
> 💡 닉네임 설정 방법: "[장치]의 [항목] 이름을 [별명]으로 지정해줘"
> ```

---

#### 2.4 24시간 데이터 트렌드 분석

> **👤 사용자**
> ```
> [장치닉네임]의 AM2301 센서 온도를 최근 24시간 동안 분석해줘
> ```

> **🤖 AI 응답**
> ```
> 📈 [[장치닉네임] > 침실온도센서] 24시간 데이터 분석
> 
> 분석 기간: 2024-07-02 14:00 ~ 2024-07-03 14:00
> 
> 📊 온도 통계:
> - 최고 온도: 29.8°C (13:45)
> - 최저 온도: 22.1°C (04:30)
> - 평균 온도: 26.3°C
> - 현재 온도: 26.8°C
> 
> 📈 트렌드 분석:
> - 00:00~06:00: 22~24°C (야간 저온 유지)
> - 06:00~12:00: 24~27°C (일출 후 상승)
> - 12:00~18:00: 27~30°C (주간 최고온)
> - 18:00~24:00: 25~27°C (야간 하강 시작)
> 
> 💡 예상: 18시 이후 온도가 25°C 이하로 하강할 것으로 보입니다.
> 에어컨을 오후 6시 이후에 끄시면 전력 절약에 도움이 됩니다.
> ```

### ⚙️ 4. Tasmota 고급 설정

#### 4.1 Telemetry 주기 변경

> **👤 사용자**
> ```
> 장치 [장치닉네임]의 상태 보고(Telemetry) 주기를 10분으로 변경해줘
> ```

> **🤖 AI 응답**
> ```
> ✅ TelePeriod 설정 완료!
> 
> 장치: [장치닉네임] (내부 ID: [장치ID])
> 이전 TelePeriod: 300초 (5분)
> 새 TelePeriod: 600초 (10분)
> 
> 💡 TelePeriod가 10분으로 변경되어 센서 데이터 수집 주기가 늘어났습니다.
> 배터리 사용 장치라면 전력 소모가 감소합니다.
> ```

---

#### 4.2 전원 복구 모드 설정

> **👤 사용자**
> ```
> 장치 [장치닉네임]의 정전 후 전원 켜짐 상태를 '이전 상태 기억'으로 설정해줘
> ```

> **🤖 AI 응답**
> ```
> ✅ 전원 복구 모드 설정 완료!
> 
> 장치: [장치닉네임] (내부 ID: [장치ID])
> 설정: PowerOnState = 3 (이전 상태 기억)
> 
> 💡 정전이 발생해도 복구 시 이전 상태로 돌아갑니다.
> ```

---

#### 4.3 타이머 설정/해제

> **👤 사용자**
> ```
> 장치 [장치닉네임]의 거실등을 켜면 30초 뒤에 자동으로 꺼지도록 타이머를 설정해줘
> ```

> **🤖 AI 응답**
> ```
> ✅ 타이머 규칙이 설정되었습니다!
> 
> 장치: [장치닉네임] (내부 ID: [장치ID])
> 조건: 거실등(POWER1)이 ON으로 변경됨
> 동작: 30초 후 자동 OFF
> 
> 적용된 Tasmota Rule:
> Rule1: ON Power1#State=1 DO RuleTimer1 30 ENDON
> Rule1: ON Rules#Timer=1 DO Power1 0 ENDON
> 
> 💡 타이머 해제 방법: "[장치닉네임]의 거실등 타이머를 해제해줘"
> ```

> **👤 사용자**
> ```
> [장치닉네임]의 거실등을 켜면 자동으로 꺼지는 타이머를 해제해줘
> ```

> **🤖 AI 응답**
> ```
> ✅ 타이머 규칙이 해제되었습니다!
> 
> 장치: [장치닉네임] (내부 ID: [장치ID])
> 해제된 규칙: 거실등 ON → 30초 후 OFF
> 
> 💡 이제 거실등을 켜도 자동으로 꺼지지 않습니다.
> ```

---

#### 4.4 Interlock (상호 잠금) 설정

> **👤 사용자**
> ```
> 장치 [장치닉네임]의 power1, power2, power3, power4을 서로 Interlock으로 설정해줘
> ```

> **🤖 AI 응답**
> ```
> ✅ Interlock 설정 완료!
> 
> 장치: [장치닉네임] (내부 ID: [장치ID])
> Interlock 그룹: [POWER1, POWER2, POWER3, POWER4]
> 
> 적용된 명령:
> Interlock 1,2,3,4
> Interlock ON
> 
> 💡 이제 POWER1이 ON이면 나머지(POWER2,3,4)는 자동 OFF 됩니다.
> 하나의 출력만 활성화되어야 하는 장치에 유용합니다.
> ```

---

#### 4.5 백로그(Backlog) 복합 명령

> **👤 사용자**
> ```
> PulseTime 절대 쓰지 말고, 반드시 Delay 명령어를 사용해서
> [장치닉네임]의 전원1을 켜고, 10초 대기한 다음 전원1을 다시 끄는 명령을
> 백로그(Backlog)로 한 번에 보내줘
> ```

> **🤖 AI 응답**
> ```
> ✅ 백로그 명령 생성 완료!
> 
> 장치: [장치닉네임] (내부 ID: [장치ID])
> 
> 전송할 명령:
> Backlog Power1 ON; Delay 100; Power1 OFF
> 
> 💡 Delay 100 = 10초 (Tasmota Delay는 0.1초 단위)
> 
> 이 명령을 그대로 Tasmota 콘솔에 붙여넣으면
> 전원1이 켜지고 10초 후 자동으로 꺼집니다.
> ```

### 🏠 5. 셔터/블라인드 제어

#### 5.1 셔터 모드 설정

> **👤 사용자**
> ```
> [장치닉네임]의 블라인드(셔터) 기능을 활성화하고, 셔터1의 제어 릴레이를 1번으로 지정해줘. 백로그로 한 번에 처리해
> ```

> **🤖 AI 응답**
> ```
> ✅ 셔터 설정 완료! (백로그 일괄 처리)
> 
> 장치: [장치닉네임] (내부 ID: [장치ID])
> 
> 전송된 백로그 명령:
> Backlog SetOption80 1; ShutterRelay1 1; ShutterMode 1
> 
> 설정 결과:
> - SetOption80 ON: 셔터 지원 활성화
> - ShutterRelay1 = 1: 셔터1에 릴레이1 할당
> - ShutterMode = 1: 차고문 모드(one relay garage mode)
> 
> 💡 이제 "[장치닉네임]의 셔터1 열어줘" 또는 "셔터1 닫아줘"로 제어 가능합니다.
> ```

---

#### 5.2 셔터 동작 시간 설정

> **👤 사용자**
> ```
> [장치닉네임]의 셔터1이 완전히 열리는 데 걸리는 시간은 12.5초로, 완전히 닫히는 데 걸리는 시간은 11.2초로 설정해줘
> ```

> **🤖 AI 응답**
> ```
> ✅ 셔터 동작 시간 설정 완료!
> 
> 장치: [장치닉네임] (내부 ID: [장치ID])
> 셔터1:
> - 개방 시간: 12.5초
> - 폐쇄 시간: 11.2초
> 
> 적용된 명령:
> ShutterOpenDuration1 125
> ShutterCloseDuration1 112
> 
> 💡 Tasmota에서 시간은 0.1초 단위로 입력됩니다.
> (12.5초 = 125, 11.2초 = 112)
> ```

### 📐 6. Tasmota Rule 직접 설정

#### 6.1 온습도 기반 Rule 생성

> **👤 사용자**
> ```
> Tasmota에서 Rules 기능으로 온습도 센서에 맞춰서 규칙을 설정하는 이벤트 트리거(Syntax) 예제를 알려줘
> ```

> **🤖 AI 응답**
> ```
> 📚 Tasmota Rule 온습도 트리거 예제
> 
> 기본 형식: ON <Trigger> DO <Action> ENDON
> 
> 1. 온도 28°C 이상 → 팬 ON
> Rule1 ON tele-AM2301#Temperature>28 DO Power1 ON ENDON
> 
> 2. 습도 60% 이상 → 제습기 ON
> Rule2 ON tele-AM2301#Humidity>60 DO Power2 ON ENDON
> 
> 3. 복합 조건 (온도 AND 습도)
> Rule3 ON tele-AM2301#Temperature>28 AND tele-AM2301#Humidity>60 DO Power1 ON ENDON
> 
> 4. 토글 동작 (ON/OFF 자동 전환)
> Rule4 ON tele-AM2301#Temperature>28 DO Power1 ON ENDON
> Rule4 ON tele-AM2301#Temperature<25 DO Power1 OFF ENDON
> 
> 💡 적용 방법:
> 1. Tasmota 콘솔에 Rule 명령어 입력
> 2. Rule1 1 (Rule 활성화)
> 3. RuleTimer1 60 (60초마다 체크)
> 
> ⚠️ 센서 이름 확인: STATUS 8 명령어로 정확한 센서명 확인
> ```

---

#### 6.2 Rule Timer 활용

> **👤 사용자**
> ```
> Tasmota의 Rule Timer 기능을 사용하고 싶어. RuleTimer1이 만료되었을 때 거실등(POWER1)을 자동으로 끄는 룰 시나리오를 설계하고 적재해줘
> ```

> **🤖 AI 응답**
> ```
> ✅ Tasmota Rule Timer 설정 완료!
> 
> 장치: [장치닉네임] (내부 ID: [장치ID])
> 
> 적용된 Rule:
> Rule1 ON Rules#Timer=1 DO Power1 0 ENDON
> 
> 활성화 명령:
> Rule1 1
> 
> 타이머 시작:
> RuleTimer1 30 (30초 후 자동 OFF)
> 
> 💡 전체 과정:
> 1. RuleTimer1 30 입력 → 30초 타이머 시작
> 2. 30초 경과 → Rules#Timer=1 이벤트 발생
> 3. Rule1 실행 → POWER1 OFF
> 
> ⚠️ RuleTimer는 1회용입니다. 다시 시작하려면 RuleTimer1 [초] 재입력
> ```

## 📊 8. AI 채팅 효율성 비교

### 동일 작업: "거실등 켜고, 온도 확인하고, 5분 후 자동 OFF 설정"

| 방식 | 조작 횟수 | 소요 시간 | 토큰 소모 |
|------|-----------|-----------|-----------|
| **대시보드** | 5~7회 클릭 | 30~60초 | - |
| **AI 채팅** | 1회 입력 | 5~10초 | ✅ 최소화 |

> **💡 복합 명령 시 AI 채팅이 더 효율적입니다!**
> 
> 여러 번의 대화 없이 **한 문장**으로 모든 작업을 처리할 수 있어
> 전체 토큰 사용량이 오히려 감소합니다.

| 데이터 분석 | **AI 채팅** | *"[장치닉네임] 24시간 온도 트렌드 분석해줘"* |

2. **간결한 표현**: 불필요한 수식어 제외
3. **닉네임 활용**: 긴 장치 ID 대신 간단한 별명 사용

## 🎯 11. 마무리

**AI 채팅은 복잡한 작업을 단순화하고, 다중 명령을 효율적으로 처리하는 강력한 도구입니다.**

### 📌 핵심 포인트

1. **단순 제어는 대시보드**, 복합 명령은 **AI 채팅** 활용
2. **한 번에 여러 명령**을 입력하면 토큰 효율성 증가
3. **별명/닉네임**을 활용하면 더 간결한 명령 가능 (장치 ID 대신 직관적인 닉네임 사용)
4. **Tasmota 고급 기능**도 자연어로 쉽게 설정 가능
5. **데이터 분석**은 AI 채팅이 가장 효과적
6. **장치 ID와 닉네임 모두 사용 가능** - 내부 식별자(ID)는 시스템용, 닉네임은 사용자용

> 💡 **프로 팁**: 
> "이 작업을 대시보드에서 하려면 몇 번 클릭해야 할까?"라고 생각되면, 
> AI 채팅으로 한 번에 처리하는 것이 더 효율적입니다!
> 
> **닉네임을 자주 활용하세요!** `[장치ID]` 대신 `거실`, `안방` 같은 닉네임을 사용하면 훨씬 자연스러운 명령이 가능합니다.
