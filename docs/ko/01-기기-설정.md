# 1. Tasmota 기기 초기 설정 (TLS 8883 & MQTT 보안 강화)

스마트 플러그, 릴레이 보드, 센서 노드 등의 IoT 장치에 Tasmota 펌웨어를 설치하고, TasMind 보안 서버망에 안전하게 연결하는 과정입니다.

---

## 1.1 펌웨어 설치 (Web Installer 활용)

> 🛠️ **준비물**: Tasmota를 설치할 스마트 기기, USB 데이터 케이블, Chrome 또는 Edge 브라우저

### Step 1. Tasmota Web Installer 접속
Chrome 또는 Edge 브라우저를 열고 [Tasmota Web Installer](https://tasmota.github.io/install/)로 이동합니다.

### Step 2. 설치 메뉴 옵션 이해하기
화면에 표시되는 주요 옵션은 다음과 같습니다. **특별한 경우가 아니라면 기본값을 그대로 두고 진행하면 됩니다.**

| 옵션 항목 | 설명 |
|-----------|------|
| **Tasmota(english)** | 펌웨어 종류 선택 (국가별 언어, 장치별, 개발 버전 선택 가능). 기본값은 영문 버전입니다. |
| **버전 선택 (날짜 표시)** | 펌웨어의 개발 및 수정 이력이 표시됩니다. 기본값은 가장 최신 버전입니다. |
| **Filter by date** | 드롭다운 메뉴로 특정 날짜의 펌웨어 버전을 선택 및 조회할 수 있습니다. |
| **Flash Speed** | 펌웨어 다운로드 및 설치 속도를 선택합니다. 기본값은 `2000000`입니다. |
| **Upload factory firmware (점선 영역)** | 직접 수정한 펌웨어 파일이나 공장 초기화 펌웨어 파일을 드래그 앤 드롭(Drag & Drop)으로 업로드할 수 있는 공간입니다. |

### Step 3. 기기 연결 및 Connect
1. USB 케이블로 기기를 PC에 연결합니다.
2. 화면의 **"Connect"** 버튼을 클릭합니다.
3. 팝업 창이 나타나면, USB에 연결된 장치 목록 중에서 **Tasmota를 설치할 대상 장치**를 선택하고 다시 **Connect** 버튼을 누릅니다.
4. 선택이 완료되면 자동으로 펌웨어 설치 화면으로 전환되며, 설치가 진행됩니다 (약 2~3분 소요).

> ⚠️ **장치가 목록에 나타나지 않나요?**
> 
> ESP32나 CH340/CH341 칩셋 기반 모듈이 보이지 않는다면, PC가 USB 포트를 올바르게 인식하지 못하는 상태입니다.  
> 아래 드라이버 중 본인의 칩셋에 맞는 것을 검색하여 PC에 설치한 후, 다시 시도하세요.
> - **CP210x 드라이버** (주로 실리콘랩 칩셋)
> - **CH34x 드라이버** (CH340, CH341 칩셋)

### Step 4. Wi-Fi 연결 설정
펌웨어 설치(삭제 및 재설치)가 완료되면, 자동으로 Wi-Fi 연결 설정 화면으로 넘어갑니다.
1. 장치를 연결할 **공유기(AP)의 Wi-Fi**를 목록에서 선택합니다.
2. 해당 Wi-Fi의 **비밀번호**를 입력하고 연결합니다.

### Step 5. 설치 완료 및 장치 IP 확인
1. Wi-Fi 연결이 성공하면 화면에 **"Visit Device"** 버튼이 활성화됩니다.
2. **"Visit Device"** 버튼을 클릭하면 해당 장치의 Tasmota 웹 설정 화면이 열립니다.
3. 이때 브라우저 주소창을 확인하세요. `192.168.0.xxx`와 같은 형식의 **IP 주소**가 표시됩니다.
   - 이 IP 주소가 바로 펌웨어가 설치된 장치의 네트워크 주소입니다.
   - 앞으로는 브라우저 주소창에 이 **IP 주소만 입력**해도 바로 Tasmota 웹 화면으로 접속할 수 있습니다.

> 💡 **장치의 IP 주소를 잊어버렸을 때는?**
> 
> 1. 다시 [Tasmota Web Installer](https://tasmota.github.io/install/)에 접속하여 **"Connect"**를 누릅니다.
> 2. 이미 Tasmota가 설치된 장치는 팝업 목록에서 별도로 표시되며, 자동으로 설정 메뉴로 이동합니다.
> 3. 다시 **"Visit Device"** 버튼을 눌러 현재 할당된 IP를 확인하거나, 여기서 Wi-Fi 설정을 변경할 수도 있습니다.

## 1.2 Wi-Fi 연결하기 (AP 모드 - 수동 설정)

> 위 1.1절의 자동 Wi-Fi 설정 과정에서 연결에 실패했거나, Wi-Fi를 변경해야 할 경우 이 방법을 사용합니다.

1. 기기가 부팅된 후, 스마트폰이나 PC의 Wi-Fi 목록에서 `tasmota-xxxxxx-xxxx` 형태의 네트워크를 찾아 접속합니다.
2. 브라우저를 열고 주소창에 `http://192.168.4.1`을 입력합니다.
3. 화면에 나타나는 집 Wi-Fi 이름(SSID)과 비밀번호를 입력하고 **저장(Save)** 버튼을 누릅니다.
4. 기기가 재부팅되며, 설정한 공유기에 정상적으로 연결됩니다.

---

## 1.3 모듈 설정 (GPIO 핀 할당)

기기가 어떤 역할(릴레이, 센서 등)을 할지 GPIO 핀을 매핑합니다.

1. 앞서 확인한 장치의 IP 주소로 브라우저에서 접속합니다 (예: `http://192.168.0.xxx`).
2. 상단 메뉴에서 **Configuration → Configure Module**로 이동합니다.
3. 기기의 실제 회로(보드)에 맞게 GPIO 핀을 설정합니다.
   - **릴레이(스위치)**: `Relay 1`, `Relay 2` 등
   - **온습도 센서**: `AM2301` (DHT22 호환)
   - **조도/가스 센서**: `ANALOG`
4. 설정이 완료되면 **저장(Save)** 버튼을 누르고 기기를 재부팅합니다.

## 1.4 MQTT 보안 설정 (가장 중요!)

TasMind는 보안을 위해 TLS 암호화 통신(8883 포트)을 필수로 사용합니다. **User와 Topic은 반드시 동일해야 합니다.**

1. **Configuration → Configure MQTT** 메뉴로 이동합니다.
2. 아래 표의 내용을 정확히 입력합니다.

| 설정 항목 | 입력값 | 비고 |
|-----------|--------|------|
| **Host** | TasMind MQTT 브로커 서버 주소 | 시스템 관리자(운영자)로부터 받은 주소를 입력 |
| **Port** | `8883` | **반드시 8883 사용!** (TLS 보안 포트, 1883 포트는 차단됨) |
| **User** | `tasmota_ai2` (예시) | 장치를 구분하는 고유 식별자 |
| **Topic** | `tasmota_ai2` (예시) | **반드시 User 필드와 동일하게 입력!** |
| **Password** | 발급받은 보안 토큰 | 장치별로 발급된 고유 비밀번호 |

> ⚠️ **User와 Topic이 불일치하면?**
> 
> - ✅ **User = Topic** → 브로커 인증 성공, 정상 연결
> - ❌ **User ≠ Topic** → ACL(Access Control List) 인증 오류로 브로커가 접속을 차단하며, 연결 재시도 루프에 빠질 수 있습니다. **두 값을 반드시 일치시키세요.**

## 1.5 기기 이름 및 별명 설정 (Friendly Name)

웹 대시보드와 앱에서 기기를 직관적으로 알아보기 위해 이름을 설정합니다.

1. **Configuration → Other** (또는 Configure Other) 메뉴로 이동합니다.
2. **Device Name** 항목에 장치의 용도를 쉽게 알 수 있는 이름을 입력합니다.  
   (예시: `1층_온실`, `공장_A구역`)
3. **Friendly Name 1 ~ N** 항목에 ON/OFF 제어가 필요한 출력 장치(릴레이, 플러그 등)의 별칭을 **한글**로 직관적으로 입력합니다.  
   (예시: `거실 온풍기`, `침실 메인조명`, `수경재배 펌프`)
4. `MQTT Enable` 옵션이 체크(활성화)되어 있는지 최종 확인하고 **저장(Save)** 합니다.

> 💡 **TasMind와의 연동 규칙**
> 
> 설정이 완료되면, 입력한 **Device Name**과 **Friendly Name**은 자동으로 TasMind 웹/앱 대시보드에 동기화되어 표시됩니다. 복잡한 ID 대신 이 이름들로 기기를 쉽게 찾고 제어할 수 있습니다.
